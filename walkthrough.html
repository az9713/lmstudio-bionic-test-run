Setting up LM Studio Bionic with a local Gemma model

Seven clicks from a blank window to an AI agent writing and running real code on your machine, no cloud calls involved. Captured step by step, screenshot by screenshot, so you can follow the exact same path.

Before you start: Bionic needs at least one model already downloaded to run anything locally. If your model list in Choose a model comes up empty, open the main LM Studio app first and pull a small model — a 4B-class model like Gemma is a good fit for most laptops.

01

Launch Bionic

Open the Bionic app. You land on a blank New tab with a single input at the center: Ask Bionic to do something. That's the home screen for every new session — nothing to configure yet.

Bionic

The starting screen — no project or model chosen yet.

02

Choose your model

Click Choose a model in the bottom-right of the composer. A panel opens listing cloud models at the top (DeepSeek V4 Pro, GLM 5.2, Kimi K2.6, Kimi K2.7 Code) and local models below, each marked with a chip icon and a GGUF tag.

Click Gemma 4 E4B in the local group. The button relabels itself Gemma 4 E4B On — confirmation the model is loaded into memory and answering entirely on this machine.

Choose a model

Local models are tagged GGUF and separated from the cloud list.

Model selected

"On" means loaded and ready — not just selected.

03

Create a new project

Click Workspace at the top of the left sidebar, then + Create new project. Name it, then choose a type: Code, for working inside a real folder with file, search, git, and shell tools, or Work, for research and document tasks. Click Create.

Workspace menu

Workspace → Create new project.

Create new project

Named, typed as Code, ready to create.

Sidebar

The project now lives in the sidebar, selected and ready.

04

Point the project at a folder

Code projects need a real working directory before they'll touch anything. Type your task into the composer, then look underneath it for Select folder in amber not set.

Click it. Windows' normal folder picker opens — browse to (or create) the folder you want Bionic working in, then click Use Folder. Select folder is replaced by your folder's name, and the send button turns solid.

Before

Task typed, but the folder isn't set — send won't do anything yet.

After

Folder name replaces "Select folder"; send lights up.

05

Send your first task

With a model On, a project selected, and a folder set, press the blue arrow (or hit Enter). We asked for:

"Create a Python file fizzbuzz.py that prints FizzBuzz from 1 to 30. Keep it simple."

06

Watch it work

The composer collapses into a session view. A Working… indicator shows while the model reasons and edits files — entirely offline, no tokens leaving the machine.

Bionic Coder Session

Local inference in progress — no network activity required.

07

Review what changed

When it finishes, Bionic shows its response, the file contents inline, and an Edited files panel listing every file touched with a line-level diff. We ran the file ourselves afterward to confirm: correct FizzBuzz output, 1 through 30.

Result

fizzbuzz.py, +12 −0, written in 4m 41s on local hardware.

Two things worth knowing

  • Send stays inert until the folder is set. For a Code project, if your first message seems to do nothing, check whether the composer still shows Select folder in amber.
  • The first load takes a moment. Switching to a local model for the first time can take 20–30 seconds while it's read into memory. Every response after that starts immediately.

Quick reference, for next time

  1. Composer shows your model name with On
  2. Project created and selected in the sidebar
  3. A folder name — not Select folder — sits under the composer
  4. Task typed, arrow clicked
  5. Edited files panel appears when it's done
